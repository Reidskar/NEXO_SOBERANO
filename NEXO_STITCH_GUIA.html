🧩 1. ¿Qué es Google Stitch?

Google Stitch es la plataforma de Google (integrada en Firebase / Google Cloud) que permite crear apps móviles y web con IA generativa. Combina Gemini AI, Firebase, Firestore, Auth, y deployment automático en un solo flujo.
¿QUÉ ES?
App Builder + IA
Genera código React Native (móvil) y React Web desde prompts. Conecta con tu backend automáticamente.
¿PARA QUÉ?
Apps NEXO en minutos
App iOS/Android y Web profesional conectada al backend FastAPI de NEXO SOBERANO con IA integrada.
¿COSTO?
Gratuito con límites
Plan Spark gratis incluye hosting, auth, Firestore. Plan Blaze (pay-as-you-go) para producción.

🔑 2. Crear cuenta y proyecto en Stitch

Paso a paso para crear la cuenta:

PASO 1
Ir a Firebase Console
Ve a console.firebase.google.com con tu cuenta Google. Click en "Agregar proyecto".
PASO 2
Nombre del proyecto
Escribe nexo-soberano-app. Acepta Google Analytics (recomendado para métricas de uso).
PASO 3
Habilitar Stitch / Gemini
En Firebase Console → "Build" → "App Hosting" → "Extensions" → buscar "Generative AI" con Gemini.
PASO 4
Configurar servicios
Activar: Authentication (Email+Google), Firestore, Storage, Functions, App Hosting.
PASO 5
Instalar Firebase CLI
En terminal VSCode: npm install -g firebase-tools luego firebase login y firebase init.
PASO 6
Vincular con VSCode
Instalar extensión "Firebase" en VSCode. Ctrl+Shift+P → "Firebase: Connect to Project" → seleccionar nexo-soberano-app.
⚠️ Google Stitch (como producto standalone) está en evolución continua. Si no aparece como "Stitch" exactamente, busca "Firebase Studio" o "Project IDX" (idx.google.com) — ambos permiten el mismo flujo con Gemini IA integrada para generar apps.

PROMPT WEB APP →

Crea una web app React (Vite + TypeScript) para NEXO SOBERANO desplegable en Firebase App Hosting. Funcionalidades: autenticación Firebase con guard de rutas protegidas, dashboard principal con métricas (Chart.js), mapa Leaflet con marcadores geoJSON de eventos de inteligencia, módulo OSINT con terminal de análisis interactiva, integración Polymarket con mercados de predicción en tiempo real, chat IA conectado a backend FastAPI, sincronización Google Drive/OneDrive con estado visual, sistema de notificaciones en tiempo real con Firebase Firestore listeners. Diseño: dark command center, misma paleta que NEXO v3. Tailwind CSS + shadcn/ui. React Router v6. TanStack Query para server state. PWA habilitado (funciona offline). Sin errores TypeScript strict. firebase.json y .firebaserc incluidos.

PROMPT BACKEND FIREBASE FUNCTIONS →

Crea Firebase Cloud Functions (Node.js TypeScript) que hagan de proxy entre la app y el backend FastAPI de NEXO SOBERANO. Funciones necesarias: /api/chat (proxy a Anthropic Claude API), /api/osint (análisis OSINT básico), /api/markets (caché de Polymarket), /api/intel (CRUD de inteligencia en Firestore), /api/sync-status (estado de sincronización). Incluir autenticación JWT con Firebase Auth, rate limiting, logging con Firebase Logger, y manejo de errores robusto. CORS configurado. Funciones scheduladas para actualizar caché de mercados cada 5 minutos.

🔌 7. Integración con el backend NEXO

Cliente API (TypeScript) — conecta Stitch/Firebase con FastAPI

// src/api/nexoClient.ts
import axios from 'axios';
import { getAuth } from 'firebase/auth';

const BACKEND = import.meta.env.VITE_NEXO_BACKEND || 'http://localhost:8000';

export const nexoApi = axios.create({ baseURL: BACKEND });

nexoApi.interceptors.request.use(async (config) => {
  const user = getAuth().currentUser;
  if (user) {
    const token = await user.getIdToken();
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

export const nexoChat = (msg: string) =>
  nexoApi.post('/api/chat', { message: msg, stream: false });

export const nexoIntel = () =>
  nexoApi.get('/api/intel');

export const nexoSyncStatus = () =>
  nexoApi.get('/api/sync/status');

🔍 8. Sistema auto-supervisor en VSCode

Instrucciones de instalación y uso:

  • Copiar nexo_autosupervisor.py en la raíz del proyecto NEXO SOBERANO
  • Copiar vscode_settings.json → .vscode/settings.json
  • Copiar vscode_tasks.json → .vscode/tasks.json
  • Abrir terminal en VSCode: python nexo_autosupervisor.py --scan
  • Para modo continuo: python nexo_autosupervisor.py --watch --fix
  • Para mejora con IA: configurar ANTHROPIC_API_KEY en .env
  • Ver reporte: python nexo_autosupervisor.py --report
  • Instalar extensión VSCode: Emerald Walk Run on Save para auto-escaneo
💡 El supervisor genera: un reporte Markdown legible + JSON detallado en reports/supervisor/. El archivo latest.md siempre tiene el estado actual del proyecto. Backups automáticos antes de cada auto-fix en .supervisor_backups/.

Qué detecta y corrige automáticamente:

CÓDIGO TIPO SEVERIDAD AUTO-FIX DESCRIPCIÓN
SV001-SV010 Seguridad CRÍTICO No eval, SQL injection, credenciales hardcodeadas
SY001 Sintaxis CRÍTICO No Errores de sintaxis Python
CX001 Complejidad MEDIO No Funciones >80 líneas
ST001 Estilo MEDIO No except desnudo
ST003 Estilo INFO SÍ print() → logging
HT005 Seguridad ALTO No innerHTML (XSS)
HT007 Estilo JS BAJO SÍ var → let/const
AC001 Accesibilidad BAJO No img sin alt

🚀 9. Deploy y publicación

Comandos de deploy:

# 1. Build web app
npm run build

# 2. Deploy a Firebase Hosting
firebase deploy --only hosting

# 3. Deploy Functions
firebase deploy --only functions

# 4. Deploy todo
firebase deploy

# 5. Ver URL de la app desplegada
firebase hosting:channel:list

# App Móvil — Build con Expo
npx eas build --platform all --profile production
npx eas submit --platform ios
npx eas submit --platform android
🔑 Variables de entorno necesarias en Firebase Functions → Secrets:
ANTHROPIC_API_KEY, NEXO_BACKEND_URL, POLYMARKET_API_URL
